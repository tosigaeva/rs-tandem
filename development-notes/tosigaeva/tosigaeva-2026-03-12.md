### Дата: 2025-03-11

**Что сделано:**

Добавила возможность отображения виджетов для выбранной темы. Компоненты `WidgetList` и `WidgetCard` для визуального представления виджетов уже были готовы, поэтому сосредоточилась на логике фильтрации и отображения.

Добавила новый виджет `All Exercises`, который позволяет пользователю практиковаться со всеми типами упражнений сразу, при этом исключая `FlipCard`. Для этого пришлось немного изменить логику фильтрации и объединения массивов виджетов.

Добавила мок-данные (widget.mock.ts) для всех типов виджетов: `Quiz`, `TrueFalse`, `CodeCompletion`, `FlipCard`, `BigONotation`.

Реализовала фильтрацию виджетов по типу через query-параметр (`?widgetType=quiz`). При выборе фильтра `all` исключается `FlipCard`. Для корректного определения фильтра добавила функцию` toWidgetFilter` в `types/widget.ts`.

Страница темы теперь рендерит либо `WidgetLis`t, либо `QuestionsRunner` в зависимости от выбранного фильтра. Для этого добавлен компонент `TopicContent`, который подгружает нужные данные и обрабатывает фильтр.

Удалены устаревшие моки `learning.question.mock` и связанные маппинги.

**Мысли:**

Сначала рассматривала вариант URL-сегмента (`/Library/1/quiz`), но отказалась от него по двум причинам:

**1** **Не хотелось менять текущую структуру папок.**
Сейчас страница темы находится в `[topicId]/page.tsx`. Если бы использовала сегмент для типа виджета, пришлось бы создавать дополнительный уровень `[widgetType]` и переносить туда логику вопросов, что усложнило бы проект.

**2** **Вопросы логически относятся к теме.**
Query-параметр (`/Library/1?widgetType=quiz`) позволяет оставлять одну страницу темы и просто менять содержимое в зависимости от выбранного фильтра.

**Итог:** query-параметр оказался более удобным и логичным решением, без лишней вложенности и изменений структуры проекта.

### Дата: 2025-03-12

Добавила пагинацию для страницы `Library`. Реализован компонент `Pagination` на основе ShadCN, с отображением номеров страниц, кнопок `Previous/Next` и активной страницы. Пагинация интегрирована с `TopicList `и применяется только к топикам, которые пользователь ещё не начал.

API `getTopicsOverview` теперь принимает параметры `page` и `limit`, возвращает массив топиков с информацией о текущей странице, общем количестве и количестве страниц (Paginated). Это позволило подгружать только нужные топики и корректно отображать навигацию по страницам.

Столкнулась с проблемой: секция с топиками, которые пользователь ещё не начал, находится внизу страницы, после блока с его текущими темами. При переходе на другую страницу список топиков прокручивался вверх, и приходилось заново скроллить вниз, что было неудобно. Сначала думала использовать якорь, но это вызывало мигание и дергание страницы. Решила проблему заменой `<a>` в `ShadCN на Link с scroll={false}` — теперь при смене страницы скролл не происходит, и пользователь остаётся в видимой области топиков.
